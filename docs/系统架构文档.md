> 版本: 1.5.2 | 最后更新: 2026-04-20

## 一、系统概述

**WeRSS（WeChat Official Account RSS Subscription Assistant）** 是一个**微信公众号内容订阅与 RSS 生成平台**。它通过自动化手段抓取微信公众号的文章内容，将其转换为标准 RSS 订阅源，使用户可以通过任意 RSS 阅读器（如 Folo、Feedly 等）订阅和阅读公众号文章。

### 核心价值

- **打破信息孤岛**：将封闭的微信公众号内容转化为开放的 RSS 标准格式
- **自动化采集**：定时自动抓取新文章，无需手动刷新
- **多端阅读**：通过 RSS 阅读器在任何设备上阅读公众号文章
- **消息推送**：新文章自动推送到钉钉/飞书/企业微信等渠道
- **分布式扩展**：级联架构支持多节点协同采集，突破单节点采集能力上限

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Python 3.13+ / FastAPI / Uvicorn |
| 前端框架 | Vue 3 / TypeScript / Vite / Arco Design |
| 数据库 | SQLAlchemy ORM / SQLite（默认）/ MySQL / PostgreSQL |
| 缓存 | Redis / 文件缓存 |
| 浏览器自动化 | Playwright / Selenium（反爬虫检测） |
| 定时调度 | APScheduler / Cron |
| 部署 | Docker / Docker Compose |

## 四、核心功能详解

### 4.1 微信公众号采集

这是系统最核心的功能，通过三种模式抓取公众号文章：

| 模式 | 实现类 | 原理 | 特点 |
|------|--------|------|------|
| **API 模式** | `MpsApi` | 调用 `/cgi-bin/appmsg` 接口 | 获取临时文章链接，速度快 |
| **Web 模式** | `MpsWeb` | 调用 `/cgi-bin/appmsgpublish` 接口 + 浏览器提取 | 获取发布链接，链接永久有效 |
| **APP 模式** | `MpsAppMsg` | 同 Web 模式接口 | 采集最新消息 |

**采集流程**：

```
WxGather.Start(mp_id)
    │
    ├── 1. 获取 Token/Cookie（从 Redis 或本地文件）
    ├── 2. 选择采集模式（Model 工厂方法）
    ├── 3. 逐页抓取文章列表
    │       └── 去重检查（HasGathered / RecordAid）
    ├── 4. 可选：抓取文章正文内容
    │       ├── web 模式：WXArticleFetcher（Playwright 移动端浏览器）
    │       └── api 模式：MpsApi（HTTP 请求）
    ├── 5. 文章入库回调（UpdateArticle）
    ├── 6. Webhook 通知（web_hook）
    └── 7. 清除 RSS 缓存
```

### 4.2 微信授权与登录

系统提供两种授权方式，通过 `server.auth_web` 配置切换：

| 方式 | 驱动类 | 原理 | 适用场景 |
|------|--------|------|---------|
| **Web 认证** | `Wx` | Playwright 打开登录页 → 截图二维码 → 等待扫码 → 提取 Cookie/Token | 需要浏览器环境 |
| **API 认证** | `WeChatAPI` | 纯 HTTP 请求获取二维码 → 轮询扫码状态 → 提取 Cookie/Token | 无浏览器环境，更轻量 |

**Token 持久化策略**：Redis（优先）+ 本地文件双写，确保重启后可恢复登录状态。

**授权过期处理**：自动检测 Token 过期 → 发送通知（钉钉/飞书/企业微信/Bark）→ 提示用户扫码重新授权。

### 4.3 RSS 订阅源生成

支持多种标准格式，兼容主流 RSS 阅读器：

| 格式 | MIME 类型 | 说明 |
|------|----------|------|
| RSS 2.0 XML | `application/rss+xml` | 最广泛兼容，支持 `content:encoded` 全文输出 |
| Atom | `application/atom+xml` | IETF 标准 |
| JSON Feed | `application/feed+json` | 现代格式 |
| 自定义模板 | 可配置 | 基于 TemplateParser 模板引擎 |

**RSS 特性**：
- 支持按公众号、按标签生成订阅源
- 支持关键词搜索过滤
- 支持全文/摘要输出
- 支持封面图 enclosure
- 支持 CDATA 包裹
- 支持 OPML 格式导出（RSS 阅读器通用导入格式）
- 文章内容缓存（避免重复抓取）

### 4.4 消息通知与 Webhook

**通知渠道**：

| 渠道 | 格式 | 配置项 |
|------|------|--------|
| 企业微信 | Markdown | `notice.wechat` |
| 钉钉 | Markdown | `notice.dingding` |
| 飞书 | Interactive Card | `notice.feishu` |
| Bark | JSON | 自动识别 URL |
| 自定义 Webhook | JSON | `notice.custom` |

**消息任务**：每个消息任务关联一组公众号，支持 Cron 定时调度，两种消息类型：
- **通知类型**（message_type=0）：渲染模板后发送到配置的 Webhook 地址
- **Webhook 类型**（message_type=1）：渲染 JSON payload 后 HTTP POST 到目标 URL

**模板引擎**：支持变量替换 `{{variable}}`、条件判断 `{% if %}`、循环 `{% for %}`、表达式计算等。

### 4.5 级联系统（分布式采集）

级联系统是 WeRSS 的分布式扩展方案，采用**父子节点架构**：

```
┌──────────────────────────────────────────────┐
│                  父节点                        │
│  ┌──────────────────────────────────────┐    │
│  │  CascadeScheduleService (定时调度)     │    │
│  │       │                               │    │
│  │  CascadeTaskDispatcher (任务分发)      │    │
│  │       │                               │    │
│  │  CascadeTaskAllocation (任务分配记录)   │    │
│  └──────────────────────────────────────┘    │
│          │ 通知              │ 认领            │
└──────────┼──────────────────┼────────────────┘
           │                  │
     ┌─────▼─────┐     ┌─────▼─────┐
     │  子节点 A   │     │  子节点 B   │
     │ CascadeSync│     │ CascadeSync│
     │ Service    │     │ Service    │
     │            │     │            │
     │ 心跳/同步   │     │ 心跳/同步   │
     │ 认领任务    │     │ 认领任务    │
     │ 执行采集    │     │ 执行采集    │
     │ 上行数据    │     │ 上行数据    │
     └────────────┘     └────────────┘
```

**核心机制**：
- **任务分发**：父节点根据 Cron 表达式定时创建任务，将公众号拆分分配给子节点
- **原子认领**：子节点通过数据库锁机制互斥认领任务，避免重复采集
- **心跳监控**：子节点定期发送心跳，父节点据此判断节点在线状态（3 分钟超时）
- **数据同步**：子节点从父节点拉取公众号和任务配置，采集完成后上行文章数据
- **负载均衡**：根据节点配额和当前负载选择最优节点分配任务

### 4.6 反爬虫引擎

系统内置了 21 项 JavaScript 反检测防护措施，确保浏览器自动化不被微信检测：

| 类别 | 防护措施 |
|------|---------|
| 自动化检测 | WebDriver 属性删除/重定义、`__playwright`/`__selenium` 痕迹清除 |
| 浏览器指纹 | Chrome 对象完整模拟、WebGL 指纹伪装（NVIDIA RTX 3060）、Canvas 噪声、AudioContext 防护 |
| 环境特征 | Navigator 属性完善（plugins/languages/platform）、Permissions API 修复 |
| 网络安全 | WebRTC 禁用（防 IP 泄露）、HTTP 头随机化 |
| 行为模拟 | 随机延迟点击、鼠标移动轨迹模拟、时区伪装 |

**UA 生成器**：按浏览器市场份额权重随机生成真实感 User-Agent，覆盖 Chrome/Safari/Firefox/Edge/Opera/QQ 等主流浏览器。

### 4.7 文章内容过滤

支持自定义 HTML 内容过滤规则，自动清理文章中的广告、推荐链接等：

| 过滤方式 | 说明 |
|---------|------|
| 移除 ID 元素 | 按 HTML ID 移除指定元素 |
| 移除 Class 元素 | 按 CSS Class 移除指定元素 |
| CSS 选择器 | 使用 CSS Selector 精确定位移除 |
| 属性过滤 | 按元素属性名和值移除 |
| 正则表达式 | 精确匹配内容并移除 |
| 通用标签移除 | 移除 script/style/comment 等标签 |

规则支持**全局规则**（所有公众号生效）和**公众号专属规则**，通过优先级控制执行顺序。

### 4.8 文章导出

支持将文章导出为多种格式：

| 格式 | 实现方式 |
|------|---------|
| Markdown | markdownify 转换 |
| DOCX | python-docx 生成 |
| PDF | reportlab / playwright PDF 导出 |
| JSON | 直接序列化 |
| CSV | 标准库 csv 模块 |

## 六、认证体系

系统支持三种认证方式，按优先级依次尝试：

```
请求到达
    │
    ├── 1. AK-SK 认证（Authorization: AK-SK {ak}:{sk}）
    │       └── 用于 API 集成场景，支持权限控制（读/写/删除/管理）
    │
    ├── 2. 级联节点认证（X-Cascade-Api-Key / X-Cascade-Api-Secret）
    │       └── 用于子节点与父节点之间的通信认证
    │
    └── 3. JWT Bearer 认证（Authorization: Bearer {token}）
            └── 用于 Web 管理端用户登录，支持角色（admin/editor/user）
```

**安全特性**：
- 密码使用 bcrypt 哈希存储
- JWT Token 支持配置过期时间（默认 3 天）
- 登录失败锁定（5 次失败后锁定 30 分钟）
- AK/SK 使用 SHA256 哈希验证
- 配置文件敏感字段自动遮蔽

## 七、任务调度架构

```
┌─────────────────────────────────────────────────────┐
│                   任务调度体系                        │
│                                                      │
│  MessageTask (数据库中的任务定义)                       │
│       │                                              │
│       ▼                                              │
│  TaskScheduler (APScheduler Cron 调度)                │
│       │                                              │
│       ▼                                              │
│  add_job() (将公众号列表加入任务队列)                    │
│       │                                              │
│       ├──► TaskQueue (主队列 - 文章采集)                │
│       │       │                                      │
│       │       ▼                                      │
│       │   do_job() (执行单个公众号采集)                  │
│       │       ├── WxGather (微信采集)                  │
│       │       ├── UpdateArticle (入库)                │
│       │       ├── web_hook (通知/Webhook)             │
│       │       └── MessageTaskTracker (进度追踪)        │
│       │                                              │
│       └──► ContentTaskQueue (内容补抓队列)              │
│               │                                      │
│               ▼                                      │
│           sync_article_content (补抓文章正文)           │
│                                                      │
│  WebSocket ──► 实时推送队列状态到前端                    │
└─────────────────────────────────────────────────────┘
```

**任务队列特性**：
- Redis 持久化（支持多进程共享状态）
- 指数退避重试（最多 3 次）
- 去重检查（同一任务不重复入队）
- 实时状态广播（WebSocket）

## 十二、核心数据流

### 文章采集完整流程

```
1. 定时调度触发（TaskScheduler / Cron）
       │
2. 加载 MessageTask，解析关联的公众号列表
       │
3. 为每个公众号创建采集任务，加入 TaskQueue
       │
4. TaskQueue 取出任务，调用 do_job()
       │
5. WxGather 初始化（加载 Token/Cookie）
       │
6. 选择采集模式（API/Web/APP），逐页抓取文章列表
       │
7. 去重检查 → 新文章入库（UpdateArticle）
       │
8. 可选：抓取文章正文（WXArticleFetcher / MpsApi）
       │
9. 应用过滤规则，清洗 HTML 内容
       │
10. Webhook 通知 / 消息推送
       │
11. 清除 RSS 缓存（下次请求时重新生成）
       │
12. 如果是级联子节点，上行数据到父节点
       │
13. WebSocket 广播任务状态更新
```
