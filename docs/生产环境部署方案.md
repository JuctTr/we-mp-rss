# We-MP-RSS 生产环境部署方案

## 一、部署方式总览

We-MP-RSS 支持多种生产环境部署方式，可根据实际场景选择：

| 部署方式 | 适用场景 | 复杂度 | 推荐度 |
|----------|---------|--------|--------|
| **Docker 单容器** | 个人使用、快速部署 | 低 | ★★★★ |
| **Docker Compose（SQLite）** | 个人/小团队、无外部数据库 | 低 | ★★★ |
| **Docker Compose 完整方案** | 生产环境、多服务 | 中 | ★★★★★ |
| **宿主机直接部署** | 定制化需求、已有基础设施 | 高 | ★★ |
| **Kubernetes 部署** | 大规模集群、高可用 | 高 | ★★★ |
| **级联分布式部署** | 大规模采集需求 | 高 | ★★★★ |

---

## 二、方案一：Docker 单容器部署（最简单）

### 适用场景
- 个人使用
- 快速体验
- 测试环境

### 部署命令
```bash
docker run -d \
  --name we-mp-rss \
  -p 8001:8001 \
  -v ./data:/app/data \
  -e USERNAME=admin \
  -e PASSWORD=your_strong_password \
  -e TZ=Asia/Shanghai \
  ghcr.io/rachelos/we-mp-rss:latest
```

### 访问
- Web 管理界面：`http://your_ip:8001`
- API 文档：`http://your_ip:8001/api/docs`

### 镜像源选择

| 镜像源 | 命令 | 适用地区 |
|--------|------|---------|
| GitHub Container Registry | `ghcr.io/rachelos/we-mp-rss:latest` | 海外 |
| Docker Hub 官方 | `rachelos/we-mp-rss:latest` | 海外 |
| 国内镜像加速 | `docker.1ms.run/rachelos/we-mp-rss:latest` | 中国大陆 |

### 注意事项
- SQLite 数据库存储在 `./data` 目录
- 适合单用户、轻量级使用
- 数据需要定期备份 `./data` 目录

## 三、方案二：Docker Compose + SQLite（轻量生产方案）

### 适用场景
- 个人或小团队
- 不需要外部数据库
- 希望使用 Compose 管理

### 1. 准备配置文件
在项目根目录创建 `.env` 文件：
```bash
USERNAME=admin
PASSWORD=your_strong_password
TZ=Asia/Shanghai
```

### 2. 部署
```bash
cd compose
docker compose -f docker-compose-sqlite.yaml up -d
```

### 3. 服务结构
```
we-mp-rss (容器)
├── 端口: 8001
├── 数据库: SQLite (./data/we_mp_rss.db)
└── 数据卷: ./data:/app/data
```

## 四、方案三：Docker Compose 完整方案（推荐生产部署）

### 适用场景
- **生产环境首选**
- 需要 MySQL 数据库
- 需要 Redis 缓存
- 需要代理解决微信 IP 封锁问题

### 1. 服务架构
```
┌─────────────────────────────────────────────┐
│               Docker Compose                 │
│                                              │
│  ┌─────────┐  ┌────────┐  ┌─────────────┐  │
│  │  MySQL   │  │ Redis  │  │  SingBox    │  │
│  │  :3306   │  │ :6379  │  │  Proxy:7890 │  │
│  └────┬─────┘  └───┬────┘  └─────┬───────┘  │
│       │            │             │          │
│  ┌────┴────────────┴─────────────┴───────┐  │
│  │           We-MP-RSS 主服务             │  │
│  │  端口: 8001                            │  │
│  │  数据库: mysql://rss_user@db-mp/...   │  │
│  │  缓存: redis://redis:6379/0           │  │
│  │  代理: http://singbox:7890            │  │
│  └───────────────────────────────────────┘  │
│                                              │
└─────────────────────────────────────────────┘
```

### 2. 配置步骤

#### 第一步：复制并配置 `.env` 文件
```bash
cd we-mp-rss
cp .env.example .env
```

编辑 `.env` 文件：
```bash
# 应用配置
USERNAME=admin
PASSWORD=your_strong_password_here  # 修改为强密码
TZ=Asia/Shanghai
AUTO_RELOAD=False  # 生产环境关闭自动重载

# MySQL 配置
MYSQL_ROOT_PASSWORD=your_mysql_root_password  # 修改为强密码
MYSQL_DATABASE=we_mp_rss
MYSQL_USER=rss_user
MYSQL_PASSWORD=your_db_password_here  # 修改为强密码

# 代理配置（如需访问微信）
PROXY_ENABLED=True
PROXY_URL=socks5://username:password@proxy.example.com:1080
```

#### 第二步：检查 compose 配置
文件位置：`compose/docker-compose.yaml`

确认以下配置项：
```yaml
services:
  we-mp-rss:
    environment:
      DB: mysql+pymysql://rss_user:pass123456@db-mp/we_mp_rss?charset=utf8mb4  # 密码需与 .env 一致
      REDIS_URL: redis://redis:6379/0
      PROXY_HTTP_URL: http://singbox:7890
```

#### 第三步：启动服务
```bash
cd compose
docker compose up -d
```

#### 第四步：验证服务
```bash
# 检查所有容器状态
docker compose ps

# 查看主服务日志
docker logs we-mp-rss

# 验证服务可访问
curl http://localhost:8001/api/docs
```

### 3. 数据持久化
```
compose/
├── db_data:/       # MySQL 数据卷
└── redis_data:/    # Redis 数据卷
./data              # 应用数据（缓存、导出文件等）
```

### 4. 常用运维命令
```bash
# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f we-mp-rss

# 重启服务
docker compose restart we-mp-rss

# 停止服务
docker compose down

# 更新镜像并重新部署
docker compose pull
docker compose up -d --force-recreate

# 备份数据库
docker exec db-mp mysqldump -u rss_user -ppass123456 we_mp_rss > backup.sql

# 恢复数据库
docker exec -i db-mp mysql -u rss_user -ppass123456 we_mp_rss < backup.sql
```

## 五、方案四：宿主机直接部署

### 适用场景
- 已有完善的服务器基础设施
- 需要深度定制
- 希望直接控制系统进程

### 1. 环境要求
- **Python** >= 3.13.1
- **Node.js** >= 20.18.3（如需编译前端）
- **Playwright** 浏览器驱动

### 2. 部署步骤

#### 第一步：安装 Python 依赖
```bash
cd we-mp-rss
pip install -r requirements.txt
```

#### 第二步：配置
```bash
cp config.example.yaml config.yaml
```

编辑 `config.yaml`：
```yaml
app_name: we-mp-rss
server:
  name: we-mp-rss
  enable_job: True
  auto_reload: False  # 生产环境关闭
  threads: 2

db: sqlite:///./data/db.db  # 或 mysql+pymysql://user:pass@host/db

secret: your_jwt_secret_key_here  # 必须设置强密钥

port: 8001

# 通知 Webhook（可选）
notice:
  dingding: ""
  wechat: ""
  feishu: ""
  custom: ""

# RSS 配置
rss:
  base_url: https://your_domain.com/
  local: False
  title: My WeChat RSS
  page_size: 30

# 安全配置
safe:
  hide_config: "db,secret,token,notice.wechat,notice.feishu,notice.dingding"
  lic_key: "YOUR_ENCRYPTION_KEY"

# 日志配置
log:
  file: /var/log/we-mp-rss.log
  level: INFO
```

#### 第三步：初始化并启动
```bash
# 初始化系统
python main.py -job True -init True
```

### 3. 使用 systemd 管理服务（推荐）

创建服务文件 `/etc/systemd/system/we-mp-rss.service`：
```ini
[Unit]
Description=We-MP-RSS WeChat RSS Service
After=network.target

[Service]
Type=simple
User=we-mp-rss
Group=we-mp-rss
WorkingDirectory=/opt/we-mp-rss
Environment=PATH=/opt/we-mp-rss/venv/bin
ExecStart=/opt/we-mp-rss/venv/bin/python main.py -job True -init True
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable we-mp-rss
sudo systemctl start we-mp-rss
sudo systemctl status we-mp-rss
```

### 4. 前端构建（如需更新前端）
```bash
cd web_ui
yarn install
yarn build
# 构建产物将输出到 ../static/ 目录
```

## 六、方案五：使用 Nginx 反向代理（生产推荐）

### 为什么需要反向代理
- **HTTPS 支持**：通过 Let's Encrypt 启用 SSL
- **域名访问**：使用域名代替 IP:端口
- **负载均衡**：多实例部署
- **静态资源缓存**：提升性能
- **安全加固**：隐藏后端端口

### 1. Nginx 配置示例

创建 `/etc/nginx/conf.d/we-mp-rss.conf`：
```nginx
server {
    listen 80;
    server_name rss.yourdomain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name rss.yourdomain.com;

    # SSL 证书配置
    ssl_certificate /etc/letsencrypt/live/rss.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/rss.yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 客户端上传大小限制
    client_max_body_size 50M;

    # 代理后端服务
    location / {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态资源缓存
    location /static/ {
        proxy_pass http://127.0.0.1:8001;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # API 路由
    location /api/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 速率限制
        limit_req zone=api burst=20 nodelay;
    }
}
```

### 2. 速率限制配置
在 `nginx.conf` 的 `http` 块中添加：
```nginx
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
```

### 3. 使用 Let's Encrypt 申请证书
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d rss.yourdomain.com
```

### 4. Docker Compose + Nginx 整合部署

创建 `docker-compose.prod.yaml`：
```yaml
services:
  we-mp-rss:
    image: ghcr.io/rachelos/we-mp-rss:latest
    container_name: we-mp-rss
    restart: unless-stopped
    env_file:
      - .env
    environment:
      DB: mysql+pymysql://rss_user:pass123456@db-mp/we_mp_rss?charset=utf8mb4
      REDIS_URL: redis://redis:6379/0
    volumes:
      - ./data:/app/data
    expose:
      - "8001"
    depends_on:
      - mysql
      - redis
    networks:
      - internal

  nginx:
    image: nginx:alpine
    container_name: we-mp-rss-nginx
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./certs:/etc/nginx/certs:ro
    depends_on:
      - we-mp-rss
    networks:
      - internal
      - external

  mysql:
    # ... MySQL 配置

  redis:
    # ... Redis 配置

networks:
  internal:
    internal: true
  external:
    internal: false
```

## 七、方案六：级联分布式部署

### 适用场景
- **大规模公众号采集需求**
- 单节点采集能力不足
- 需要横向扩展采集能力

### 架构说明
```
                    ┌─────────────────┐
                    │   父节点 (父)    │
                    │  - 任务分发      │
                    │  - 数据汇总      │
                    │  - API 服务      │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
       ┌──────┴──────┐ ┌───┴────┐ ┌───────┴──────┐
       │  子节点 1   │ │ 子节点2│ │   子节点 3   │
       │  - 采集任务 │ │-采集任务│ │  - 采集任务   │
       │  - 数据上报 │ │-数据上报│ │  - 数据上报   │
       └─────────────┘ └────────┘ └──────────────┘
```

### 1. 父节点配置
```bash
# .env 或 config.yaml
CASCADE_ENABLED=True
CASCADE_NODE_TYPE=parent
CASCADE_API_KEY=your_api_key
CASCADE_API_SECRET=your_api_secret
```

### 2. 子节点配置
```bash
# .env 或 config.yaml
CASCADE_ENABLED=True
CASCADE_NODE_TYPE=child
CASCADE_PARENT_API_URL=http://parent_node_ip:8001
CASCADE_API_KEY=your_api_key
CASCADE_API_SECRET=your_api_secret
CASCADE_SYNC_INTERVAL=300  # 同步间隔（秒）
CASCADE_HEARTBEAT_INTERVAL=60  # 心跳间隔（秒）
CASCADE_TASK_POLL_INTERVAL=30  # 任务拉取间隔（秒）
```

### 3. 部署命令
```bash
# 父节点
docker run -d \
  --name we-mp-rss-parent \
  -p 8001:8001 \
  -v ./data:/app/data \
  -e CASCADE_ENABLED=True \
  -e CASCADE_NODE_TYPE=parent \
  -e CASCADE_API_KEY=your_key \
  -e CASCADE_API_SECRET=your_secret \
  ghcr.io/rachelos/we-mp-rss:latest

# 子节点（可在不同服务器上部署多个）
docker run -d \
  --name we-mp-rss-child1 \
  -p 8001:8001 \
  -v ./data:/app/data \
  -e CASCADE_ENABLED=True \
  -e CASCADE_NODE_TYPE=child \
  -e CASCADE_PARENT_API_URL=http://parent_ip:8001 \
  -e CASCADE_API_KEY=your_key \
  -e CASCADE_API_SECRET=your_secret \
  ghcr.io/rachelos/we-mp-rss:latest
```

## 八、方案七：Kubernetes 部署

### 适用场景
- 大规模集群
- 需要高可用
- 已有 K8s 基础设施

### 1. Deployment 配置 `we-mp-rss-deployment.yaml`
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: we-mp-rss
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: we-mp-rss
  template:
    metadata:
      labels:
        app: we-mp-rss
    spec:
      containers:
        - name: we-mp-rss
          image: ghcr.io/rachelos/we-mp-rss:latest
          ports:
            - containerPort: 8001
          env:
            - name: DB
              value: "sqlite:///data/we_mp_rss.db"
            - name: USERNAME
              value: "admin"
            - name: PASSWORD
              valueFrom:
                secretKeyRef:
                  name: we-mp-rss-secret
                  key: password
            - name: TZ
              value: "Asia/Shanghai"
          volumeMounts:
            - name: data
              mountPath: /app/data
          resources:
            requests:
              memory: "512Mi"
              cpu: "250m"
            limits:
              memory: "1Gi"
              cpu: "500m"
      volumes:
        - name: data
          persistentVolumeClaim:
            claimName: we-mp-rss-data
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: we-mp-rss-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
---
apiVersion: v1
kind: Service
metadata:
  name: we-mp-rss
spec:
  selector:
    app: we-mp-rss
  ports:
    - port: 8001
      targetPort: 8001
  type: ClusterIP
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: we-mp-rss
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  tls:
    - hosts:
        - rss.yourdomain.com
      secretName: we-mp-rss-tls
  rules:
    - host: rss.yourdomain.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: we-mp-rss
                port:
                  number: 8001
```

### 2. 部署
```bash
kubectl apply -f we-mp-rss-deployment.yaml
```

## 九、代理配置方案

### 为什么需要代理
微信可能会限制数据中心 IP 访问，需要配置代理来正常采集。

### 方案一：SingBox 代理（推荐）
```bash
# .env 中配置
PROXY_ENABLED=True
PROXY_URL=socks5://username:password@proxy.example.com:1080
```

SingBox 支持两种代理协议：
- **SOCKS5**：`socks5://user:pass@host:port`
- **HTTP**：`http://user:pass@host:port`

### 方案二：HTTP 代理
```yaml
# config.yaml
proxy:
  enabled: True
  http_url: http://127.0.0.1:7890  # 或 socks5://127.0.0.1:1080
```

### 方案三：Deno Deploy 代理
```yaml
# config.yaml
proxy:
  enabled: True
  deno_url: https://your-proxy.deno.dev
```

## 十、生产环境安全建议

### 1. 必须配置的安全项

| 配置项 | 建议 | 说明 |
|--------|------|------|
| `SECRET_KEY` | 强随机字符串 | JWT 加密密钥，必须设置 |
| `PASSWORD` | 强密码 | 管理员密码至少 12 位 |
| `MYSQL_PASSWORD` | 强密码 | 数据库密码至少 12 位 |
| `REDIS_SERVER_PASSWORD` | 强密码 | Redis 认证密码 |
| `SAFE_HIDE_CONFIG` | 默认配置 | 隐藏敏感配置信息 |

### 2. 网络安全
- **启用 HTTPS**：使用 Let's Encrypt 或商业证书
- **限制端口暴露**：只暴露 80/443，内部服务不暴露端口
- **配置防火墙**：只允许必要端口访问
- **使用内网**：Docker Compose 中使用内部网络隔离

### 3. 数据安全
- **定期备份**：数据库和 `data` 目录每日备份
- **加密存储**：敏感数据加密存储
- **权限控制**：限制容器用户权限
- **日志审计**：启用访问日志和操作日志

### 4. 容器安全
- **不挂载 Docker Socket**：避免容器逃逸
- **使用只读文件系统**：只挂载必要的目录
- **限制资源**：设置 CPU 和内存限制
- **定期更新镜像**：使用最新安全补丁

## 十一、备份与恢复

### 1. SQLite 备份
```bash
# 备份数据库
cp data/db.db data/db_backup_$(date +%Y%m%d).db

# 或使用 SQLite 的 .backup 命令
sqlite3 data/db.db ".backup 'data/db_backup.db'"
```

### 2. MySQL 备份
```bash
# 备份
docker exec db-mp mysqldump -u rss_user -ppass123456 we_mp_rss > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i db-mp mysql -u rss_user -ppass123456 we_mp_rss < backup_20260419.sql
```

### 3. 完整备份脚本
```bash
#!/bin/bash
BACKUP_DIR="/backup/we-mp-rss"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份数据库
docker exec db-mp mysqldump -u rss_user -ppass123456 we_mp_rss > $BACKUP_DIR/db_$DATE.sql

# 备份数据目录
tar czf $BACKUP_DIR/data_$DATE.tar.gz ./data

# 备份配置文件
cp .env $BACKUP_DIR/env_$DATE
cp config.yaml $BACKUP_DIR/config_$DATE.yaml

# 保留最近 30 天的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: $DATE"
```

## 十四、常见问题

### Q1: 启动后无法访问微信文章
**解决方案**：配置代理，推荐使用 SingBox
```bash
PROXY_ENABLED=True
PROXY_URL=socks5://user:pass@proxy.example.com:1080
```

### Q2: 数据库连接失败
**解决方案**：检查数据库连接字符串格式
```bash
# SQLite
DB=sqlite:///data/db.db

# MySQL
DB=mysql+pymysql://user:password@host:3306/dbname?charset=utf8mb4

# PostgreSQL
DB=postgresql://user:password@host:5432/dbname
```

### Q3: 内存占用过高
**解决方案**：调整线程数和缓存配置
```yaml
server:
  threads: 1  # 降低线程数
cache:
  enabled: True
  ttl: 3600  # 设置缓存过期时间
```

### Q4: 如何升级版本
```bash
# Docker Compose 方式
docker compose pull
docker compose up -d --force-recreate

# 单容器方式
docker stop we-mp-rss
docker rm we-mp-rss
docker pull ghcr.io/rachelos/we-mp-rss:latest
docker run -d --name we-mp-rss -p 8001:8001 -v ./data:/app/data ghcr.io/rachelos/we-mp-rss:latest
```
